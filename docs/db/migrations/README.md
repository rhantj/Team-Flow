# 이 디렉터리는 운영 DB에 스키마 변경을 수동으로 적용한 이력입니다

이 저장소는 Flyway 등 자동 마이그레이션 도구를 쓰지 않습니다. 여기 있는 `NNN_*.sql`
파일들은 운영 DB(Supabase/OCI)에 `psql`로 직접 적용한 스키마 변경 이력입니다. 번호가
겹치는 파일이 여러 개 있는 것도(예: `006_users_profile_fields.sql`과
`006_task_results.sql`) 여러 기능 브랜치가 각자 독립적으로 다음 번호를 붙이다 병합되며 생긴
결과입니다 — 실제 적용 순서를 이 번호만으로는 알 수 없습니다.

**새 스키마 변경은 다음 번호로 여기에 파일을 추가하고, 배포 전에 운영 DB에 직접
`psql`로 실행하세요** — 자동으로 적용되는 경로가 없으므로 이 실행 자체가 배포 절차의
일부입니다. `db/init/`(`backend_spring/src/main/resources/db/init/`)에도 같은 내용의 파일을
함께 추가해야 완전히 새로 만드는 DB(로컬 최초 기동 등)에도 반영됩니다. 자세한 절차는
[`App/DEPLOY_OCI.md`](../../../App/DEPLOY_OCI.md) 8절을 참고하세요.

011(`011_drop_legacy_field.sql`)만은 예외입니다 — `users.field` 컬럼 정리는 구버전 인스턴스를
깨뜨릴 수 있는 파괴적 변경이라 의도적으로 자동 배포 경로에 넣지 않았고, 파일 내
`RENAME`/`DROP` 구문도 주석 처리해뒀습니다. 여전히 `App/DEPLOY_OCI.md` 8-1절의 체크리스트를
사람이 확인한 뒤 수동으로 주석을 해제해 한 번만 실행합니다.
